Agents are systems that use AI models to make decisions and execute tasks autonomously. They integrate with external APIs, execute custom functions, and maintain access to your data sources to handle complex workflows.

Think of agents as digital assistants with persistent context, they remember previous interactions, maintain consistent behavior patterns, and have access to specific tools and capabilities for their designated roles.

Built on Cloudflare's infrastructure, agents are stateful classes that run on Durable Objects. They handle HTTP requests, WebSocket connections, task scheduling, AI model integration and more.

> **Note**: Stateful agents require Durable Object migrations in your `wrangler.jsonc` (e.g., `new_sqlite_classes`) for proper state management.

## Quick start

```sh
npm create cloudflare@latest my-agent -- --template=cloudflare/agents-starter
cd my-agent && npx wrangler dev
```

Or add to an existing project:

```sh
npm install agents
```

## Your first agent

Create a simple joke agent that remembers previous interactions:

<TypeScriptExample>

```ts
import { Agent } from "agents";

export class JokeAgent extends Agent {
	initialState = {
		jokesShared: 0,
		favoriteTypes: [],
	};

	async onMessage(connection, message) {
		const response = await this.ai.run(`
			You are a friendly joke-telling agent.
			Previous jokes shared: ${this.state.jokesShared}
			User message: "${message}"

			If they ask for a joke, tell them one.
			If they mention a joke type they like, remember it.
			Be conversational and fun!
		`);

		connection.send(response);

		// Update state based on interaction
		const isJokeRequest = message.toLowerCase().includes("joke");
		await this.setState({
			jokesShared: isJokeRequest
				? this.state.jokesShared + 1
				: this.state.jokesShared,
		});
	}
}

export default {
	fetch: JokeAgent.serve(),
};
```

</TypeScriptExample>

This agent remembers how many jokes it has shared and adapts its responses based on conversation history.

## Building your first agent

Follow these steps to create an agent that can autonomously handle tasks:

### 1. Define agent identity and tools

```ts
export class CustomerServiceAgent extends Agent {
	initialState = {
		name: "CustomerServiceAgent",
		role: "Customer support specialist",
		conversationHistory: [],
		availableTools: ["search_docs", "create_ticket", "escalate"],
	};
}
```

### 2. Add decision-making logic

```ts
async onMessage(connection, message) {
  // Agent analyzes request and decides on actions
  const analysis = await this.ai.run(`
    As ${this.state.name}, analyze this customer request: "${message}"

    Available tools: ${this.state.availableTools.join(', ')}

    Decide what actions to take and respond appropriately.
  `);

  // Execute chosen actions
  const response = await this.executeActions(analysis, message);
  connection.send(response);
}
```

### 3. Implement persistent memory

```ts
async executeActions(analysis, originalMessage) {
  // Update conversation history
  await this.setState({
    conversationHistory: [
      ...this.state.conversationHistory,
      { message: originalMessage, timestamp: Date.now() }
    ]
  });

  // Agent remembers context for future interactions
  return analysis;
}
```

### 4. Deploy and test

```bash
npm run dev
# Your agent is now running with autonomous decision-making
```

> **Testing locally**: Use `wrangler dev` or `vitest` to test against Durable Objects and Agents locally.

Your agent can now autonomously decide on sequences of actions, maintain memory across conversations, and consistently apply its role and tools.

## Core capabilities

- Persistent state with automatic synchronization
- WebSocket connections with real-time messaging
- HTTP request handling
- Task scheduling with cron expressions
- Email integration
- AI model integration via `this.ai.run()` backed by Cloudflare AI Gateway or your provider
- React hooks for frontend integration

## Architecture

Each agent instance runs on a [Durable Object](/durable-objects/) and handles multiple concurrent connections. State changes are automatically persisted and synchronized to connected clients.

Agents can call external APIs, schedule future tasks, and communicate with other agents through defined interfaces.
